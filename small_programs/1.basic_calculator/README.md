# プログラム概要

Calculator.java は　計算機の基本的な機能をもっているプログラムです。  
主な五つの演算子で構成されています。
- 加算 (+)
- 減算 (-)
- 乗算 (*)
- 除算 (/)
- 剰余計算 (%)

# プログラムの流れ
1.プログラムを実行すると、操作番号のメニューが表示されます。<br>
2.ユーザーは行いたい演算に対応する操作番号をコンソールから入力します。<br>
3.その後、その演算に必要な2つの数値を入力します。<br>
4.入力された値をもとに計算処理を行い、結果をコンソールに出力します。<br>
5.出力後は、次の計算を行うために再度操作番号のメニューを表示します。<br>

# 操作番号（1～８）

- 各演算には操作番号1～6が割り当てられています。
- 計算を終了する場合は操作番号8
- 計算メニューを再表示する場合は操作番号7と定めています。

# 4つの非機能要件を意識した設計・開発
本プログラムは、4つの視点（非機能要件）を意識して設計・開発を行っています。<br>
具体的には、以下の4つの観点に基づいています。

### 1.安定性の視点
安定性の視点とは、**正常時だけでなく、異常時においてもプログラムが停止せず動作し続けること**を指します。<br><br>
本プログラムでは、ユーザー入力によるエラーを想定し、異常終了を防ぐために厳密な入力チェックを行っています。  
例えば、除算や剰余の場合はは**0.0入力禁止**、操作番号は**1～８しか**受け付けません。  
そして**操作番号は整数**で決まっているため、小数の入力も禁止です。  
また、計算機のプログラムなので**文字列はもちろん禁止**にしております。<br> <br> 
また、本プログラムにおける終了は以下の**2つ**に限定しています。<br> 
&nbsp;&nbsp;i.ユーザーの操作による**正常終了**（　操作番号8　）  
&nbsp;&nbsp;ii.シャットダウンや電源遮断などによる**強制終了**  

### 2.使う人の視点
使う人の視点とは、**エンドユーザーにとって使いやすいプログラムを実現すること**です。<br>
どれほど多機能であっても、ユーザーにとって分かりにくいプログラムは価値が低いと考えています。<br><br> 
そのため、本プログラムではユーザーの操作性や視認性に配慮しています。<br> 
例えば、コンソール上で**文字を整えて見やすく**表示したり、**操作番号７によるメニューの再表示機能**を設けています。<br> 
また、無効な入力があった場合には、**有効な入力内容を分かりやすく説明した上で再入力**を促しています。<br> 
さらに、2番目の数値を入力する際には、**直前に入力した1番目の数値も表示することで、現在の入力状況を把握しやすく**しています。<br><br>
このように、ユーザーの立場に立った設計により、直感的で使いやすいプログラムを実現しています。<br> 

### 3.作る人の視点
作る人の視点とは、**将来においても分かりやすく、他の技術者が見ても理解しやすいプログラムを実現すること**です。<br>
プログラムは一度作って終わりではなく、機能追加や修正などに応じて継続的に変更されるものであり、またチーム開発では他の技術者がコードを扱うことも想定されます。<br><br>
そのため、本プログラムでは**コードの可読性と保守性**に配慮しています。<br>
具体的には、各関数の上に**処理内容を説明するコメント**を記述し、クラス名・変数名・関数名についても、他の技術者が見て直感的に理解できるよう**簡潔で分かりやすい命名**を行っています。<br><br>
このように、将来の修正や他者による開発を見据えた設計により、保守しやすいプログラムを実現しています。<br>

### 4.効率性の視点
効率性の視点とは、**無駄のない処理を行い、プログラムの性能や資源の使用効率を高めること**です。<br>
同じプログラムであっても、コードの重複や不要なリソースの放置は、処理効率の低下や管理の複雑化につながると考えています。<br><br>
そのため、本プログラムでは効率的な構造を意識しています。<br>
具体的には、同じ処理が複数箇所に存在する場合は**一つの関数にまとめることでコードの共通化**を図り、無駄を削減しています。<br>
また、**使用後のリソースは適切に解放すること**で、不要な負荷を防ぎ、全体のパフォーマンス向上につなげています。<br><br>
このように、処理の効率性と資源管理を意識した設計により、無駄のないプログラムを実現しています。<br>

# プログラムのサンプル入力・出力
```text
例 1：加算
計算機の操作番号(整数)を入力してください : 1
[ _ ] + [ x ] : 5
[ 5.0 ] + [ _ ] : 3
[ 5.0 ] + [ 3.0 ] = 8.0

例 2：整数除算（商のみ）
計算機の操作番号(整数)を入力してください : 4
[ _ ] / [ x ] : 7
[ 7.0 ] / [ _ ] : 2
[ 7.0 ] / [ 2.0 ] = 3

例 3：剰余計算
計算機の操作番号(整数)を入力してください : 6
[ _ ] % [ x ] : 7
[ 7.0 ] % [ _ ] : 3
[ 7.0 ] % [ 3.0 ] = 1
```
